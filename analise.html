1. Diferença entre HTML e CSS

O que é HTML?

HTML HyperText Markup Language é a linguagem de marcação responsável pela estrutura e o conteúdo de uma página web. Com HTML, definimos o que existe na página: títulos, parágrafos, imagens, links, tabelas, formulários, etc. Cada elemento é representado por uma tag. 

<!-- Exemplo: estrutura HTML básica -->
<h1>Título Principal</h1>
<p>Este é um parágrafo de texto.</p>
<img src="foto.jpg" alt="Minha foto">
<a href="contato.html">Ir para Contato</a>

O que é CSS?

CSS Cascading Style Sheets é a linguagem de estilos responsável pela aparência e apresentação visual dos elementos HTML. Com CSS, controlamos cores, fontes, tamanhos, espaçamentos, layouts e animações. 

/* Exemplo: CSS estilizando o título acima */
h1 {
  color: #0d9488;
  font-size: 2rem;
  text-align: center;
}

Como funcionam juntos?

HTML e CSS se complementam: HTML cria o "esqueleto" da página (o quê existe), e CSS define o "visual" (como aparece). O navegador lê o HTML para construir a estrutura e, em seguida, aplica as regras CSS para renderizar o estilo final.

Analogia: Se uma página web fosse uma casa, o HTML seria a planta e a construção (paredes, portas, janelas) e o CSS seria a decoração e pintura (cores, móveis, iluminação).
HTML define:
  • Estrutura da página
  • Hierarquia do conteúdo
  • Semântica (o que cada parte significa)
  • Elementos: títulos, textos, imagens, links...
CSS define:
  • Cores e fundos
  • Tipografia (fonte, tamanho, peso)
  • Espaçamentos (margin, padding)
  • Layout (Flexbox, Grid) e animações

2. Exemplos de Propriedades CSS

Propriedade: color

O que faz: Define a cor do texto de um elemento.

Como afeta: Altera a cor das letras/texto exibido na tela. Aceita valores em nome de cor, hexadecimal, RGB e HSL.

h1 {
  color: #0d9488;      /* Hexadecimal */
}

p {
  color: rgb(45, 55, 72); /* RGB */
}

a:hover {
  color: tomato;          /* Nome de cor */
}
Visual: O texto do título aparece em verde-teal, o parágrafo em cinza escuro, e links ao passar o mouse ficam vermelhos.

Propriedade: font-size

O que faz: Define o tamanho da fonte de um elemento.

Como afeta: Controla quão grande o texto aparece. Pode usar unidades absolutas (px) ou relativas (rem, em, %).

body {
  font-size: 16px;    /* Tamanho base */
}

h1 {
  font-size: 2.5rem;  /* 2.5x o tamanho base = 40px */
}

.caption {
  font-size: 0.85em;  /* 85% do elemento pai */
}
Visual: Títulos aparecem grandes e proeminentes, enquanto legendas aparecem menores que o texto normal, criando hierarquia visual.

Propriedade: margin

O que faz: Controla o espaçamento externo ao redor de um elemento (fora da borda).

Como afeta: Cria espaço entre elementos, afastando-os uns dos outros na página.

.card {
  margin: 20px;              /* 20px em todos os lados */
}

.hero-title {
  margin-top: 0;
  margin-bottom: 16px;    /* Apenas embaixo */
}

.container {
  margin: 0 auto;           /* Centraliza horizontalmente */
}
Visual: Cards têm espaço entre si. O título da seção tem espaço abaixo para separar do conteúdo. O container é centralizado na página.

3. Três Formas de Aplicar CSS

  • CSS Inline
  • CSS Interno
  • CSS Externo

1. CSS Inline (Na linha)

Como funciona: O CSS é escrito diretamente dentro do atributo style de um elemento HTML. As propriedades se aplicam apenas àquele elemento específico.

<!-- Exemplo de CSS Inline -->
<h1 style="color: #0d9488; font-size: 2rem; text-align: center;">
  Título com Estilo Inline
</h1>

<p style="margin: 0; background-color: #f0fff4; padding: 10px;">
  Parágrafo com fundo verde claro.
</p>
Vantagens:
  • Aplicação imediata e específica
  • Alta especificidade (sobrescreve outras regras)
  • Útil para testes rápidos ou estilos únicos
Desvantagens:
  • Mistura estrutura e estilo no mesmo arquivo
  • Dificulta manutenção (necessário editar cada tag)
  • Não é reutilizável
  • Não respeita o princípio de separação de responsabilidades

2. CSS Interno (Embedded / Incorporado)

Como funciona: O CSS é escrito dentro de um bloco <style> no <head> do documento HTML. As regras se aplicam a toda a página.

<!-- Exemplo: CSS Interno no <head> -->
<!DOCTYPE html>
<html>
<head>
  <style>
    body {
      font-family: Arial, sans-serif;
      background-color: #f7fafc;
    }

    h1 {
      color: #0d9488;
      text-align: center;
    }

    .destaque {
      background: yellow;
      padding: 4px 8px;
    }
  </style>
</head>
<body>
  <h1>Título Estilizado</h1>
  <p>Texto com <span class="destaque">palavra destacada</span>.</p>
</body>
</html>
Vantagens:
  • Tudo em um único arquivo (fácil compartilhar)
  • Permite uso de seletores e reutilização dentro da página
  • Adequado para páginas únicas ou protótipos
Desvantagens:
  • Não é compartilhável entre múltiplas páginas
  • Aumenta o tamanho do arquivo HTML
  • Dificulta manutenção em projetos grandes

3. CSS Externo (Arquivo separado)

Como funciona: O CSS é escrito em um arquivo separado (.css) e vinculado ao HTML via tag <link> no <head>. Esta é a forma mais recomendada para projetos reais.

<!-- No arquivo index.html -->
<head>
  <link rel="stylesheet" href="assets/css/main.css">
  <link rel="stylesheet" href="assets/css/custom.css">
</head>
/* No arquivo assets/css/main.css */
body {
  font-family: Roboto, sans-serif;
  font-size: 16px;
  color: #272829;
}

a {
  color: var(--accent-color);
  text-decoration: none;
}

Exemplo real do arquivo assets/css/main.css do template iPortfolio usado neste portfólio.

Vantagens:
  • Separação clara de responsabilidades (HTML = estrutura, CSS = estilo)
  • Reutilizável em múltiplas páginas
  • Arquivo CSS pode ser cacheado pelo navegador (performance)
  • Facilita manutenção e trabalho em equipe
Desvantagens:
  • Requer requisição HTTP extra ao servidor
  • Página pode piscar brevemente (FOUC) antes do CSS carregar
  • Requer organização de arquivos e pastas

Comparativo das 3 Formas

FormaOnde é escritoEscopoRecomendado para
Inline Atributo style="" no elemento Apenas o elemento Ajustes pontuais, e-mails HTML
Interno Bloco <style> no <head> Toda a página Protótipos, páginas únicas
Externo Arquivo .css separado Múltiplas páginas Projetos reais (mais recomendado)

4. Identificação de Seletores no Template iPortfolio

Os trechos abaixo foram extraídos diretamente do arquivo assets/css/main.css do template iPortfolio (BootstrapMade), utilizado como base deste portfólio.

4.1 Seletores Simples

Seletor de Elemento (Type Selector)

Seleciona todos os elementos HTML de um determinado tipo.

/* Seleciona todos os elementos 'a' (links) */
a {
  color: var(--accent-color);
  text-decoration: none;
  transition: 0.3s;
}

Seletor de Classe (Class Selector)

Seleciona elementos que possuem um determinado atributo class. Prefixado com .

/* Seleciona todos os elementos com class="header" */
.header {
  position: fixed;
  top: 0;
  left: 0;
  width: 300px;
  overflow-y: auto;
  z-index: 997;
}

Seletor de ID

Seleciona o elemento único com o id especificado. Prefixado com #

/* Seleciona o elemento com id="preloader" */
#preloader {
  position: fixed;
  inset: 0;
  z-index: 999999;
  background: var(--background-color);
  transition: all 0.6s ease-out;
}

Seletor de Grupo (Group Selector)

Aplica o mesmo estilo a múltiplos seletores separados por vírgula.

/* Seleciona h1, h2, h3, h4, h5 e h6 ao mesmo tempo */
h1,
h2,
h3,
h4,
h5,
h6 {
  color: var(--heading-color);
  font-family: var(--heading-font);
}

Seletor Universal

Seleciona todos os elementos. Geralmente usado com combinadores ou para reset de estilos.

/* Seletor universal — aparece no CSS gerado pelo Bootstrap, vendored */
*,
*::before,
*::after {
  box-sizing: border-box; /* do bootstrap.min.css incluído */
}

4.2 Pseudo-classes

:hover — Estado ao passar o mouse

/* Link muda de cor ao passar o mouse */
a:hover {
  color: color-mix(in srgb, var(--accent-color), transparent 25%);
}

/* Ícone social na sidebar fica colorido ao hover */
.header .social-links a:hover {
  color: var(--contrast-color);
  background: var(--accent-color);
}

:focus — Estado com foco (teclado/clique)

/* Menu de navegação em foco */
.navmenu a,
.navmenu a:focus {
  color: var(--nav-color);
  padding: 15px 10px;
  font-size: 16px;
  display: flex;
  align-items: center;
}

/* Campo de e-mail do contato em foco fica com borda colorida */
.contact .php-email-form input[type=email]:focus {
  border-color: var(--accent-color);
}

:last-child — Último filho

/* Último item do currículo não tem padding inferior */
.resume .resume-item:last-child {
  padding-bottom: 0;
}

4.3 Combinadores

Combinador Descendente (espaço)

Seleciona elementos que são descendentes (qualquer nível) de outro elemento.

/* Seleciona img dentro de .profile-img dentro de .header */
.header .profile-img img {
  margin: 15px auto;
  display: block;
  width: 120px;
  border-radius: 50%;
}

Combinador Adjacente (+)

Seleciona o elemento imediatamente após outro elemento irmão.

/* 'li+li' seleciona cada 'li' que vem logo após outro 'li' */
.page-title .breadcrumbs ol li+li {
  padding-left: 10px;
}

.page-title .breadcrumbs ol li+li::before {
  content: "/";
  display: inline-block;
  padding-right: 10px;
}

4.4 Pseudo-elementos

::before — Insere conteúdo antes do elemento

/* Cria o spinner animado do preloader */
#preloader:before {
  content: "";
  position: fixed;
  top: calc(50% - 30px);
  left: calc(50% - 30px);
  border: 6px solid #ffffff;
  border-color: var(--accent-color) transparent var(--accent-color) transparent;
  border-radius: 50%;
  width: 60px;
  height: 60px;
  animation: animate-preloader 1.5s linear infinite;
}

::after — Insere conteúdo depois do elemento

/* Cria a linha decorativa abaixo do título da seção */
.section-title h2:after {
  content: "";
  position: absolute;
  display: block;
  width: 50px;
  height: 3px;
  background: var(--accent-color);
  left: 0;
  bottom: 0;
}

/* Cria o ponto decorativo nos itens do currículo */
.resume .resume-item::before {
  content: "";
  width: 16px;
  height: 16px;
  border-radius: 50px;
  left: -9px;
  background: var(--background-color);
  border: 2px solid var(--accent-color);
}

::placeholder — Estiliza o texto de placeholder dos inputs

/* Texto de placeholder nos campos do formulário de contato */
.contact .php-email-form input[type=text]::placeholder,
.contact .php-email-form input[type=email]::placeholder,
.contact .php-email-form textarea::placeholder {
  color: color-mix(in srgb, var(--default-color), transparent 70%);
}

4.5 Seletores de Atributo

[type=text] e [type=email] — Seleção por valor de atributo exato

/* Seleciona inputs com type="text" e type="email" no formulário */
.contact .php-email-form input[type=text],
.contact .php-email-form input[type=email],
.contact .php-email-form textarea {
  font-size: 14px;
  padding: 10px 15px;
  box-shadow: none;
  border-radius: 0;
  color: var(--default-color);
  background-color: var(--surface-color);
}

/* Botão de submit */
.contact .php-email-form button[type=submit] {
  color: var(--contrast-color);
  background: var(--accent-color);
  border-radius: 50px;
  padding: 10px 30px;
}

[data-aos-delay] — Seleção pela presença de atributo

/* Seleciona qualquer elemento que tenha o atributo data-aos-delay */
[data-aos-delay] {
  /* usado pelo AOS (Animate On Scroll) para atrasar animações */
}

No HTML do template, este atributo aparece assim:

<div class="container" data-aos="fade-up" data-aos-delay="100">
  <!-- conteúdo animado com 100ms de atraso -->
</div>

Conclusão

Este documento apresentou os conceitos fundamentais de HTML e CSS, demonstrando como essas duas tecnologias trabalham juntas para criar páginas web estruturadas e visualmente atraentes.
